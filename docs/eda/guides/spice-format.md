# SPICE Format: Hyper Explanation

## What is SPICE?

**SPICE = Simulation Program with Integrated Circuit Emphasis**

It's a text file that describes a circuit so a computer can simulate it.

---

## The Simplest SPICE File

```spice
* This is a comment (starts with *)
V1 node1 0 DC 5
R1 node1 node2 1000
.end
```

**That's it.** That's a valid SPICE file.

---

## Line by Line Explanation

```spice
* My first circuit
```
- `*` = Comment (ignored by simulator)

```spice
V1 node1 0 DC 5
```
- `V1` = Voltage source named "V1"
- `node1` = Positive terminal connects to "node1"
- `0` = Negative terminal connects to ground (always called "0")
- `DC 5` = It's 5 volts DC

```spice
R1 node1 node2 1000
```
- `R1` = Resistor named "R1"
- `node1` = One terminal
- `node2` = Other terminal
- `1000` = 1000 ohms (1kΩ)

```spice
.end
```
- End of file

## SPICE Component Syntax

### Voltage Source
```spice
Vname positive_node negative_node DC value
```
Examples:
```spice
V1 vdd 0 DC 1.8        * 1.8V power supply
VIN input 0 DC 0.5     * 0.5V input signal
```

### Resistor
```spice
Rname node1 node2 value
```
Examples:
```spice
R1 a b 1000            * 1kΩ
R2 vdd out 10k         * 10kΩ (can use k suffix)
RLOAD out 0 1meg       * 1MΩ (can use meg suffix)
```

### Current Source
```spice
Iname positive_node negative_node DC value
```
Example:
```spice
I1 0 node1 DC 1m       * 1mA current source
```

## Your FeCIM Crossbar in SPICE

### The Physical Crossbar:
```
     V0    V1    V2    V3   ← Input voltages (word lines)
      │     │     │     │
WL0 ──●─────●─────●─────●──
      │     │     │     │
WL1 ──●─────●─────●─────●──
      │     │     │     │
WL2 ──●─────●─────●─────●──
      │     │     │     │
     BL0   BL1   BL2   BL3  ← Output currents (bit lines)
      
● = FeFET cell (acts like a resistor with conductance G)
```

### The SPICE Translation:
```spice
* FeCIM Crossbar SPICE Netlist
* Generated by Module 6

* === POWER SUPPLY ===
.param VDD = 1.8
VDD vdd 0 DC {VDD}

* === WORD LINE DRIVERS (Inputs) ===
* These apply input voltages to each row
VWL0 wl0 0 DC 0.9      * Row 0 input = 0.9V
VWL1 wl1 0 DC 0.45     * Row 1 input = 0.45V
VWL2 wl2 0 DC 0.0      * Row 2 input = 0V

* === BIT LINE LOADS (for measuring output current) ===
* Small resistors to ground for current sensing
RBL0 bl0 0 1k
RBL1 bl1 0 1k
RBL2 bl2 0 1k

* === FeFET CELLS (as resistors) ===
* R_row_col connects wl[row] to bl[col]
* Resistance = 1/Conductance

* Row 0 cells
R_0_0 wl0 bl0 14285.71    * G=70μS → R=14.3kΩ
R_0_1 wl0 bl1 33333.33    * G=30μS → R=33.3kΩ
R_0_2 wl0 bl2 10000.00    * G=100μS → R=10kΩ

* Row 1 cells
R_1_0 wl1 bl0 20000.00    * G=50μS → R=20kΩ
R_1_1 wl1 bl1 100000.00   * G=10μS → R=100kΩ
R_1_2 wl1 bl2 25000.00    * G=40μS → R=25kΩ

* Row 2 cells
R_2_0 wl2 bl0 50000.00    * G=20μS → R=50kΩ
R_2_1 wl2 bl1 12500.00    * G=80μS → R=12.5kΩ
R_2_2 wl2 bl2 16666.67    * G=60μS → R=16.7kΩ

* === ANALYSIS ===
.op                        * DC operating point analysis

* === CONTROL (ngspice specific) ===
.control
run
print i(RBL0) i(RBL1) i(RBL2)
.endc

.end
```

## How the Math Works

### At Each Cell:
```
Current through cell = Voltage × Conductance
        I = V × G

Or equivalently:
        I = V / R    (since R = 1/G)
```

### At Each Bit Line (Column):
```
Total current = Sum of all currents from that column

I_BL0 = V_WL0 × G_0_0 + V_WL1 × G_1_0 + V_WL2 × G_2_0
```

**This IS matrix-vector multiplication!**

```
┌         ┐   ┌           ┐   ┌      ┐
│ I_BL0   │   │ G00 G10 G20│   │ V_WL0│
│ I_BL1   │ = │ G01 G11 G21│ × │ V_WL1│
│ I_BL2   │   │ G02 G12 G22│   │ V_WL2│
└         ┘   └           ┘   └      ┘

  Output    =    Weights    ×   Input
 (currents)   (conductances)  (voltages)
```

## SPICE Number Formats

| Suffix | Meaning | Example |
|--------|---------|---------|
| `T` | 10¹² (tera) | `1T` = 1,000,000,000,000 |
| `G` | 10⁹ (giga) | `1G` = 1,000,000,000 |
| `meg` | 10⁶ (mega) | `1meg` = 1,000,000 |
| `k` | 10³ (kilo) | `1k` = 1,000 |
| `m` | 10⁻³ (milli) | `1m` = 0.001 |
| `u` | 10⁻⁶ (micro) | `1u` = 0.000001 |
| `n` | 10⁻⁹ (nano) | `1n` = 0.000000001 |
| `p` | 10⁻¹² (pico) | `1p` = 0.000000000001 |
| `f` | 10⁻¹⁵ (femto) | `1f` = 0.000000000000001 |

Examples:
```spice
R1 a b 10k        * 10,000 ohms
R2 a b 4.7meg     * 4,700,000 ohms
C1 a b 100p       * 100 picofarads
V1 a 0 DC 1.8     * 1.8 volts
I1 0 a DC 1m      * 1 milliamp
```

## Analysis Commands

### DC Operating Point (What you need)
```spice
.op
```
Calculates all voltages and currents at steady state.

### Transient Analysis
```spice
.tran 1n 100n
```
Simulates from 0 to 100ns with 1ns steps.

### AC Analysis
```spice
.ac dec 10 1k 1G
```
Frequency sweep from 1kHz to 1GHz.

## ngspice Control Block

```spice
.control
run                           * Run the simulation
print v(node1)               * Print voltage at node1
print i(R1)                  * Print current through R1
plot v(out)                  * Plot voltage (opens window)
wrdata output.csv v(out)     * Save to CSV file
.endc
```

## Your Module 6 SPICE Export Function

```go
// This is what your spice.go does:

func GenerateSPICE(mapping *CrossbarMapping, vdd float64) string {
    var sb strings.Builder
    
    // Header
    sb.WriteString("* FeCIM Crossbar SPICE Netlist\n")
    sb.WriteString("* Generated by Demo 6\n\n")
    
    // Power
    sb.WriteString(fmt.Sprintf(".param VDD = %.2f\n", vdd))
    sb.WriteString("VDD vdd 0 DC {VDD}\n\n")
    
    // Word lines (inputs)
    sb.WriteString("* Word Line Drivers\n")
    for row := 0; row < numRows; row++ {
        sb.WriteString(fmt.Sprintf("VWL%d wl%d 0 DC 0\n", row, row))
    }
    
    // Bit line loads (outputs)
    sb.WriteString("\n* Bit Line Loads\n")
    for col := 0; col < numCols; col++ {
        sb.WriteString(fmt.Sprintf("RBL%d bl%d 0 1k\n", col, col))
    }
    
    // FeFET cells as resistors
    sb.WriteString("\n* FeFET Cells\n")
    for _, cell := range mapping.Cells {
        resistance := 1e6 / cell.Conductance  // Ω from μS
        sb.WriteString(fmt.Sprintf("R_%d_%d wl%d bl%d %.2f\n",
            cell.Row, cell.Col, cell.Row, cell.Col, resistance))
    }
    
    // Analysis
    sb.WriteString("\n.op\n")
    sb.WriteString(".control\n")
    sb.WriteString("run\n")
    for col := 0; col < numCols; col++ {
        sb.WriteString(fmt.Sprintf("print i(RBL%d)\n", col))
    }
    sb.WriteString(".endc\n")
    sb.WriteString(".end\n")
    
    return sb.String()
}
```

## How to Run Your SPICE File

```bash
# 1. Generate the SPICE file (your tool does this)
# Output: fecim_array.sp

# 2. Run ngspice
ngspice -b fecim_array.sp

# -b = batch mode (no interactive window)
```

## Summary

| Concept | SPICE Syntax |
|---------|--------------|
| Comment | `* this is ignored` |
| Voltage source | `Vname node+ node- DC value` |
| Resistor | `Rname node1 node2 value` |
| Ground | Always node `0` |
| Analysis | `.op` (DC), `.tran` (time) |
| End file | `.end` |
